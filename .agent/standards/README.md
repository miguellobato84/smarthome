# Standards Guide

This folder contains repository-specific standards for how we build and review code in this project.
Codex reads these files to understand what "good" looks like here, and humans can use them as a lightweight team agreement.
If something in this repo feels unusual or easy to get wrong, this is a good place to document it.

## How to use this folder
- Start with `index.yml` to see which standards exist and when to use them.
- Review these standards during onboarding and after major project changes.
- If you notice repeated mistakes or ambiguity, consider adding a standard for that topic.
- Aim for short standards with four parts: principles, rules, examples, and validation.

## Keeping it healthy
- When you add a new standard file, also add it to `index.yml`.
- If a standard is outdated, update or remove it so guidance stays trustworthy.
- Prefer repo-specific conventions over generic framework advice.
